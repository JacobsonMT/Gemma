Implementation

We figure some folks out there might be curious about how we created Gemma from a software engineering standpoint; the following section also gives us an opportunity to credit the numerous commercial tools and free open source projects that Gemma benefits from.

Data model

The Gemma data model has its own page.

Languages

Gemma is implemented primarily entirely in Java™ 5. The major exceptions are numerous flavors of XML (various configuration files and UML model), HTML, Javascript, and CSS (all part of the web front end).

The other exceptions to Java are a bridge to Jim Kent's BLAT (implemented in C, using JNI) that probably isn't worth the effort, and some analyses that use the R programming language (using a Java client to RServe; not used in the current version of Gemma). There is also one lonely Perl script used to make test files from GEO SOFT files.

A large segment of the Gemma Java codebase is generated from a UML model, using Andromda.

Developer tools

Windows XP, Eclipse IDE 3.2, MagicDraw 9.5 (Community or Standard edition), MyEclipse, YourKit profiler, Cygwin.

Build process and code management

Gemma builds are managed by Maven. We run a Continuum server for continuous builds. Bugzilla is used for bug-tracking, and CVS for version control.

Software stack

Gemma uses a range of runtime software technologies, all free and/or open source. A high-level overview:

  • Apache HTTPD 2.x
  • Tomcat 5.0.x
  • Java server pages.
  • Spring 1.2.8 (soon to go to 2.0)
  • Hibernate 3.2.x
  • MySQL 5.x

We were strongly influenced by Appfuse and many traces of Appfuse can be found in the Gemma codebase.

Note that while Gemma is a J2EE application, we do not use Entity Java Beans.

Hardware

The Gemma service you are using now is running on several AMD64-based servers running RedHat Linux.
