Help for search in Gemma

Search syntax

Search terms can be free text (" liver"), gene symbols (" ACAT1") or other accession numbers for specific experiments or probes. Here are some more complicated examples to explain query logic:

  • female cortex - Union of items matching either terms; alterantively you can use female OR cortex .
  • female AND cortex - Inteserction of items matching terms. Note that 'AND' must be capitalized.
  • female -cortex - Exclusion.
  • "cerebral cortex" - search for a phrase.
For detailed information on the supported query syntax see this page, which refers to ' Lucene', the search engine employed by Gemma.

Search scope

Besides the term(s) you want to search for, you must choose the search scope. Assuming you have cookies enabled in your browser, Gemma will remember your settings. It's a good idea to restrict searches to just the items you want to avoid longer waits.

The available search scopes are:

  • Genes, by official symbol, alias, or name. Examples: mGluR5 or metabotropic glutamate receptor.
  • Sequences - The nucleotide sequences used on arrays, often referred to by Genbank identifiers identifier. Examples: AA486221. You can also retrieve sequences via associated genes or probes.
  • Experiments - An expression experiment, searchable by text or the accession. For example GSE60 or liver.
  • Platforms - Expression analysis platforms or 'array designs', such as " HG-U133A".
  • Probes - Components of array platforms, usually referred to by the manufacturer's id, the id in GEO, or a Genbank accession. For example, 1450202_at

Search results

Each category of search result is presented separately in the table. A link is provided to a detailed view of each result items. If available, the text that your query matched will be shown below the results (hide/show the details using the button on the top of the results table).

If the query you ran returned many results, you may find 'Find in results' useful. Type here to restrict your view.

Note that results are limited to 500 items. For some queries, the number of results may be limited even more; if you don't see what you are looking for, try a more specific query.

Common tasks

Find probes for a gene

The best way to do this is to first locate the gene, and follow the link provided to the details page for the genes. From there click on the "probes" link to view all probes for that gene.

The alternative, of searching for 'probes' with the gene symbol, will yield less precise results.

Technical details

Gemma actually employs three different types of searches and combines the results.

  • Database search - The query term entered is searched against fields in the Gemma database for exact matches. This is how queries for Genbank identifiers are handled.
  • Full text search - Gemma indexes free text such as PubMed abstracts associated with data setts or descriptive text from GEO entries. These queries use Lucene through a Compass interface.
  • Structured annotation search - Search terms are looked for in ontologies using LARQ. Entities in Gemma which are annotated with the returned terms (or subclasses of the returned terms) are returned after the logic of the user's query is applied. For example, ' cortex AND lung' will only return datasets which are annotated with an ontology term related to 'cortex' and also annotated with a term related to 'lung'. (By related, meaning either the term or a subclass; thus 'cortex' also returns 'hippocampus').
